## MCP json 구성 기준

MCP 서버를 붙일 때 전송 방식은 크게 STDIO와 SSE가 있다.
STDIO는 로컬 프로세스의 stdin/stdout으로 통신한다.
SSE는 HTTP endpoint로 스트리밍 연결을 만든다.

## STDIO 구성 예시

jar 파일 실행 예시:

```json
"my-mcp-calculator": {
  "command": "java",
  "args": [
    "-jar",
    "/Users/bonjugi/IdeaProjects/my-mcp-servers/my-mcp-calculator/build/libs/my-mcp-calculator-1.0.0.jar",
    "--spring.profiles.active=stdio"
  ]
}
```

Docker 실행 예시:

```json
"my-mcp-calculator": {
  "command": "docker",
  "args": [
    "run",
    "-i",
    "--rm",
    "bonjugi/my-mcp-calculator:latest",
    "--spring.profiles.active=stdio"
  ]
}
```

## SSE 구성 예시

```json
"my-mcp-calculator": {
  "command": "npx",
  "args": [
    "-y",
    "mcp-remote",
    "http://localhost:8088/sse"
  ]
}
```

## 필드 메모

- `command`: 실행할 바이너리
- `args`: 실행 인자 (순서 중요)
- `--spring.profiles.active=stdio`: 예시에서는 stdio 모드로 띄우기 위한 플래그
- `mcp-remote`: HTTP SSE를 MCP로 중계하는 브릿지

## 정리

로컬/CI 환경에서는 STDIO가 단순하다.
서비스 형태로 띄우거나 원격 연결이 필요하면 SSE가 편하다.
