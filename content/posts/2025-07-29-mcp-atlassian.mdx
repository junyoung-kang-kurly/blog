## mcp-atlassian 로컬 실행하기

mcp-atlassian을 분석하려고 한다.
디버깅을 위해 로컬에서 실행해 보자.
여러 가지로 편의한 **streamable-http** 방식으로 기동하는 것이 좋다.

## 사전 준비

프로젝트마다 조금씩 다르니 README 기준으로 맞추면 된다.

1. 파이썬 실행 환경 준비 (venv, uv, poetry 중 하나)
2. 의존성 설치 (pyproject.toml 또는 requirements.txt 확인)
3. 인증/토큰 관련 환경변수 설정 (README에 나온 이름 그대로)

## 왜 streamable-http인가

stdio는 IDE나 호스트 프로세스에 직접 붙어야 해서 디버깅이 번거롭다.
streamable-http는 포트로 띄워두고, 재시작이나 테스트가 훨씬 편하다.
로컬에서 연결 상태를 눈으로 확인하기도 쉽다.

### 1. 소스코드 체크아웃

- git: https://github.com/sooperset/mcp-atlassian

### 2. 메인 클래스 경로 확인

메인 클래스는 아래 경로에 있다.

```
~/mcp-atlassian/src/mcp_atlassian/__init__.py
```

### 3. transport 수정

`main` 함수는 `transport` 시스템 변수를 받는다.
streamable-http로 실행하려면 transport를 수정한다.
